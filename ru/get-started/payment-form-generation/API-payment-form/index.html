Платежная форма по API

Использование API позволяет настраивать гибкие и кастомизированные платежные решения для вашего сайта. В частности, можно выбрать режим работы – через платежную страницу Interkassa или режим прямой оплаты.

Оплата через платежную страницу Interkassa​

Нажав кнопку оплаты, покупатель попадает сначала на платежную страницу Interkassa – это страница, где отображены все доступные способы оплаты. Выбирает, как ему удобнее рассчитаться за покупку, а затем переходит на страницу соответствующей платежной системы для ввода своих реквизитов.

Режим прямой оплаты​

Позволяет пропустить этап с платежной страницей: с вашего сайта клиент сразу переходит на страницу оплаты платежной системы. К примеру, покупатель хочет оплатить через электронный кошелек Advcash. Тогда на вашем сайте он выбирает 'Оплатить через Advcash' и сразу попадает на страницу с формой для оплаты кошельком.

Настройка приема платежей по API​

API доступен для всех мерчантов по умолчанию. Для его активации добавьте IP-адрес в поле IP-фильтр в настройках личного аккаунта.

IP фильтр в настройках аккаунта

Прием платежей​

Для начала приема платежей просто скопируйте платежную форму. Стандартная форма с обязательными параметрами выглядит так:

index.html
<form name="payment" method="post" action="https://sci.interkassa.com/" accept-charset="UTF-8">
  <input type="hidden" name="ik_co_id" value="51237daa8f2a2d8413000000"/>
  <input type="hidden" name="ik_pm_no" value="ID_4233"/>
  <input type="hidden" name="ik_am" value="1.44"/>
  <input type="hidden" name="ik_cur" value="uah"/>
  <input type="hidden" name="ik_desc" value="Payment Description"/>
  <input type="submit" value="Pay">
</form>

Настройка параметров платежной формы​

Вы можете добавить дополнительные параметры к платежной форме, влияющие на доступные способы оплаты, срок действия счета, язык платежной страницы и прочее. За подробностями обратитесь к разделу 'API'.

Использование режима прямой оплаты​

Этот режим предназначен для приема платежей через Interkassa с перенаправлением клиента на шлюз платежной системы непосредственно во время оплаты. Это позволяет вашему серверу в фоновом режиме получать данные формы HTML, предназначенные для оплаты через выбранный платежный шлюз.

Чтобы использовать этот режим, используйте интерфейс JSON SCI через параметр формы платежного запроса – ik_int со значением `json'.
